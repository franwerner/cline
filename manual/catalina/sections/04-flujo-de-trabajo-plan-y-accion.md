## Flujo de trabajo: Plan y Acción

Catalina trabaja en dos modos complementarios. El modo Plan sirve para razonar y acordar qué hacer; el modo Acción sirve para llevarlo a cabo. Ambos se alternan desde el selector situado junto al campo de entrada.

### Modo Plan

En modo Plan, Catalina analiza la petición, hace preguntas si lo necesita y propone un plan de trabajo antes de modificar nada. Es el modo recomendado para tareas que no estén claras desde el principio o que afecten a varios archivos.

En este modo Catalina:

- Examina el contexto disponible (archivos mencionados, problemas, terminal).
- Discute el enfoque con la persona usuaria.
- Propone los pasos que seguiría, sin aplicarlos todavía.

### Modo Acción

En modo Acción, Catalina ejecuta los cambios acordados: crea o edita archivos, ejecuta comandos y, si está permitido, utiliza el navegador. Cada acción que modifique el proyecto o el entorno se presenta para su aprobación.

### Alternar entre modos

1. Localizar el selector de modo junto al campo de entrada.
2. Seleccionar **Plan** para razonar o **Acción** para ejecutar.

Es habitual empezar en modo Plan para acordar el enfoque y, una vez claro, cambiar a modo Acción para aplicarlo.

### Aprobación de los pasos

Durante una tarea, Catalina solicita aprobación antes de cada acción relevante:

| Situación | Qué se aprueba |
| --- | --- |
| Edición de archivo | Crear o modificar un archivo del proyecto. |
| Ejecución de comando | Lanzar un comando en el terminal. |
| Uso del navegador | Abrir o consultar una página (si está permitido). |

La persona usuaria puede aceptar, rechazar o ajustar lo propuesto en cada paso, manteniendo así el control sobre el resultado.
